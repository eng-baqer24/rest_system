# تقرير المشروع — موقع مطعم Renaissance الفاخر

## ملخص تنفيذي

تم تصميم وتنفيذ موقع ويب لمطعم فاخر (Fine Dining) تحت اسم **Renaissance**، مع التركيز على تحويل الزائر إلى حاجز طاولة عبر واجهة أنيقة وبسيطة وتجربة حجز سريعة. التصميم مستوحى من المرجع المرفق (خلفية داكنة، لمسات ذهبية، مساحات بيضاء، أقسام Hero ونبذة وأطباق مميزة وآراء عملاء وCTA للحجز).

---

## ما تم إنجازه

### 1. الإعداد التقني

- **Next.js 16** مع App Router و TypeScript.
- **Tailwind CSS v4** للتصميم.
- **shadcn/ui** مع مكونات: Button, Card, Input, Label, Calendar, Dialog, Select, Textarea.
- **Prisma** مع schema لجدول الحجوزات (`Reservation`) وجداول قائمة الطعام (`DishCategory`, `Dish`) للاستخدام المستقبلي.
- **Supabase** كعميل جاهز في `src/lib/supabase.ts` (للاستخدام لاحقاً في المصادقة أو التخزين).
- إعداد **Next.js** لدعم الصور من `images.unsplash.com`.

### 2. الهوية والتصميم

- **الأسلوب**: فاخر، بسيط، أنيق — خلفية داكنة (كحلي/أسود)، لون ثانوي ذهبي/بيج للعناوين والأزرار.
- **الخطوط**: دعم الخطوط العربية (Amiri للعناوين، Noto Sans Arabic للنص) عبر Google Fonts؛ العناوين بـ `font-serif` والنص بـ sans-serif.
- **RTL**: الموقع بالكامل `dir="rtl"` و `lang="ar"`.
- **المساحات البيضاء**: تباعد واضح بين الأقسام لشعور فاخر وتجنب الازدحام البصري.

### 3. الصفحة الرئيسية (Homepage)

- **Hero Section**: صورة كاملة العرض للمطعم، عنوان ذهبي «تجربة طعام فاخرة لا تنسى»، عبارة ترحيب، زر «احجز طاولة».
- **نبذة عن المطعم**: قصة قصيرة + صورة الشيف، زر «اكتشف المزيد» يصل لصفحة «من نحن».
- **أطباق مميزة**: 3 أطباق (بايس المحار، ستيك فيليه، حلوى الشوكولاتة) مع صور ووصف.
- **آراء العملاء**: اقتباس واحد مع اسم عميل ونقاط تنقل (مكان لعدة آراء لاحقاً).
- **دعوة للحجز (CTA)**: عنوان «احجز طاولتك الآن» + زر «احجز الآن».

### 4. نظام الحجز (الميزة الأساسية)

- **صفحة الحجز** `/booking` على خطوتين:
  1. **الخطوة 1**: اختيار التاريخ (تقويم)، الوقت (قائمة أوقات ثابتة)، عدد الأشخاص (1–10). زر «التالي».
  2. **الخطوة 2**: الاسم، البريد الإلكتروني، رقم الهاتف (مطلوبة)، ملاحظات اختيارية (مناسبة خاصة، حساسية). زر «تأكيد الحجز».
- **لا يُطلب إنشاء حساب** — إتمام الحجز بالبيانات الأساسية فقط.
- **صفحة التأكيد** `/booking/confirm?id=...`: رسالة «تم استلام طلب الحجز» مع رقم الطلب وزر العودة للرئيسية.
- **API** `POST /api/reservations`: يستقبل البيانات ويحفظها عبر Prisma في جدول `Reservation` (يتطلب قاعدة بيانات فعّالة).

### 5. صفحة قائمة الطعام (Menu)

- تقسيم واضح: **مقبلات، أطباق رئيسية، حلويات، مشروبات** (بيانات ثابتة في `src/data/menu.ts`).
- كل صنف: صورة، اسم، وصف قصير، سعر (إن وُجد).
- قسم **قائمة التذوق** في أسفل الصفحة كنص تشجيعي للاستفسار والحجز.
- الاعتماد على محتوى وصور داخل الموقع وليس PDF فقط.

### 6. صفحة من نحن (About)

- رؤية المطعم، فلسفة الشيف، قصة العلامة التجارية.
- صور: ديكور المطعم، الشيف أثناء العمل.
- زر «احجز طاولة» في الأسفل.

### 7. صفحة التواصل والمعلومات (Contact)

- **الموقع**: عنوان نصي + رابط «عرض على الخريطة» + iframe خريطة Google Maps (إحداثيات تجريبية).
- **ساعات العمل**: نص واضح (أحد–خميس، جمعة–سبت).
- **رقم الهاتف**: قابل للنقر.
- **سياسة اللباس**: لباس أنيق، تجنب الرياضي.
- **المواقف**: موقف متوفر، خدمة صف السيارات عند الطلب.

### 8. تجربة الهاتف المحمول (Mobile)

- تصميم متجاوب (هوامش وأعمدة تتكيف مع الشاشة).
- أزرار كبيرة (مثل «احجز طاولة» في الهيدر وفي الأسفل).
- زر **«احجز طاولة» ثابت** في أسفل الشاشة أثناء التصفح (`StickyBookButton`) مع `pb-24 md:pb-28` للمحتوى حتى لا يُغطى المحتوى.
- نصوص واضحة وحجم خط مناسب.

### 9. الهيكل العام

- **Header**: شعار Renaissance، روابط (الرئيسية، قائمة الطعام، من نحن، تواصل)، زر «احجز طاولة».
- **Footer**: شعار، روابط سريعة، موقع وساعات وهاتف، أيقونات فيسبوك وإنستغرام، حقوق النشر.

## المشاكل والحلول

### 1. اسم المجلد والمشروع

- **المشكلة**: مجلد المشروع «rest system» يحتوي مسافة ولا يقبله `create-next-app` كاسم حزمة.
- **الحل**: إنشاء المشروع داخل مجلد فرعي باسم `renaissance` واستخدامه كمشروع الموقع.
- **المسار الصحيح**: تشغيل جميع أوامر المشروع (`npm install`, `npm run dev`, `npx prisma …`) **من داخل مجلد `renaissance` فقط**؛ لا يُشغّل شيء من جذر «rest system» حتى لا يختلط المسار.

### 2. قاعدة البيانات والحجز

- **المشكلة**: واجهة الحجز تعتمد على Prisma وقاعدة PostgreSQL؛ في حال عدم وجود `DATABASE_URL` أو عدم تنفيذ `prisma db push`/`migrate`، طلب الحجز سيفشل بخطأ 500.
- **الحل**: إعداد `.env.example` وملف README مع خطوات واضحة: نسخ `.env`، إضافة `DATABASE_URL` (مثلاً من Supabase)، ثم `npx prisma generate` و `npx prisma db push`. للمطور: التأكد من تشغيل المشروع مع قاعدة بيانات فعّالة قبل اختبار الحجز.

### 3. التقويم والخطوة الأولى من الحجز

- **ملاحظة**: الأوقات المعروضة حالياً ثابتة (`TIME_SLOTS`) وليست مستمدة من أوقات متاحة فعلية حسب التاريخ. تحسين مستقبلي: API يُرجع الأوقات المتاحة حسب اليوم من قاعدة الحجوزات.

### 4. الصور

- **ملاحظة**: الصور من Unsplash (روابط خارجية). في الإنتاج يُفضّل رفع الصور على Supabase Storage أو CDN واستخدام روابط دائمة خاصة بالمطعم.

### 5. Supabase

- **الحالة**: العميل مُعدّ في `src/lib/supabase.ts`؛ لم يُستخدم بعد في المصادقة أو التخزين. يمكن لاحقاً ربط تسجيل الدخول أو رفع الصور عبر Supabase.

### 6. قائمة الطعام والبيانات

- **الحالة**: قائمة الطعام تعتمد على ملف بيانات ثابت `src/data/menu.ts`. يمكن لاحقاً نقل الأصناف والفئات إلى قاعدة البيانات (جداول `DishCategory` و `Dish`) وبناء صفحة القائمة ديناميكياً.

### 7. useSearchParams وصفحة التأكيد

- **المشكلة**: Next.js يطلب أن تكون الصفحات التي تستخدم `useSearchParams()` داخل حدّ Suspense حتى لا تفشل الـ pre-render.
- **الحل**: فصل المحتوى الذي يقرأ `searchParams` في مكوّن `ConfirmContent` ولفّه بـ `<Suspense>` مع fallback «جاري التحميل...» في صفحة `/booking/confirm`.

## كيفية التشغيل والاختبار

1. الدخول لمجلد المشروع: `cd renaissance`
2. تثبيت الحزم: `npm install`
3. إعداد البيئة: `cp .env.example .env` وتعديل `DATABASE_URL`
4. توليد عميل Prisma: `npx prisma generate`
5. إنشاء الجداول: `npx prisma db push` (أو `prisma migrate dev`)
6. تشغيل الخادم: `npm run dev`
7. فتح المتصفح: `http://localhost:3000`

لاختبار الحجز يجب أن تكون قاعدة البيانات متصلة ومجدولة (Prisma).

## الخلاصة

تم تنفيذ موقع مطعم فاخر يتوافق مع المواصفات المطلوبة: هوية بصرية داكنة وذهبية، صفحة رئيسية بأقسام Hero ونبذة وأطباق مميزة وآراء وCTA، نظام حجز بخطوتين دون إجبار على إنشاء حساب، صفحة قائمة طعام منظمة، صفحة «من نحن» وصفحة تواصل مع خريطة وساعات وعنوان وهاتف وسياسة لباس ومواقف. التجربة متجاوبة مع التركيز على الهاتف وزر حجز ثابت. المشروع جاهز للتشغيل بعد ربط قاعدة البيانات وإكمال إعداد Prisma كما هو موضح أعلاه.
